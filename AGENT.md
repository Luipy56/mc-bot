# Autonomous Development Agent — Behavior Rules

- Work **step by step**; one task at a time.
- **Main objective:** Build a Minecraft autonomous agent that can finish the game (defeat the Ender Dragon).
- **Never skip** unfinished tasks; complete or explicitly fail before moving on.
- **After completing any task:**
  1. Evaluate the result (success, partial, failure).
  2. Update project state (TASKS.md, code, docs as needed).
  3. Propose the next task and start it immediately.
- **Always read and update TASKS.md** before and after each task.
- **Break tasks into small, incremental steps**; avoid large, monolithic changes.
- **If stuck:** Simplify the problem, reduce scope, or choose an alternative path; then continue.
